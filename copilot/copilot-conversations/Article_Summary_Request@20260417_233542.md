## 核心概念

**观察者模式**是一种行为设计模式，定义了一种**订阅机制**，当发布者（Publisher）的状态发生变化时，能自动通知所有已订阅的观察者（Subscribers）。

## 问题背景

以「顾客关注商店新产品」为例：
- 顾客反复去商店检查 → 浪费时间
- 商店向所有顾客发通知 → 浪费资源打扰无关顾客

## 解决方案

```
发布者 ──订阅/取消订阅──▶ 订阅者列表
    │
    └── 事件触发 → 遍历列表 → 调用 update()
```

- **发布者**：维护订阅者列表，管理订阅/退订，事件发生时通知订阅者
- **订阅者**：实现统一接口（如 `update()`），响应通知

## 主要优点

- **开闭原则**：新增订阅者无需修改发布者代码
- **运行时动态绑定**：订阅者可随时加入或离开

## 主要缺点

- 订阅者通知顺序是随机的

## 与其他模式的关系

| 模式 | 特点 |
|------|------|
| 责任链 | 请求沿链传递直到被处理 |
| 命令 | 发送者与请求者单向连接 |
| 中介者 | 通过中介者间接通信 |
| 观察者 | 接收者动态订阅/取消订阅 |

> 中介者可通过观察者实现（中介者作发布者，其他组件作订阅者）。
