## 项目简介

女娲系统旨在**提炼思维框架而非复制人物**。通过结构化调研和多维度分析，将人物的认知操作系统（心智模型、决策启发式、表达风格、价值观、反模式）转化为可运行的 Claude Code Skill。

安装方式: `npx skills add alchaincyf/nuwa-skill`

使用方式: "蒸馏一个[人名]" 或使用预置 Skill（如"用芒格的视角帮我分析"）

## 预置 Skills

13 个人物 Skill + 1 个主题 Skill，包含:
- Paul Graham, Elon Musk, Steve Jobs, Charlie Munger, Naval Ravikant, Andrej Karpathy 等

## 执行框架（5大阶段 + 双Agent精炼）

### Phase 0: 入口分流与需求澄清
- **直接路径**：明确人名/主题 → 确认聚焦方向、用途、是否更新现有Skill、是否有本地素材
- **诊断路径**：模糊需求 → 定位核心需求维度 → 推荐2-3个候选（已有Skill优先）→ 用户选择

诊断时识别需求维度（决策、表达、商业、教学、批判思维等），追问不超过2轮。候选展示需包含核心镜片、为什么适合、局限性。

### Phase 0.5: Skill目录创建
创建标准结构：
```
.claude/skills/[person-name]-perspective/
├── SKILL.md
├── scripts/（字幕下载/清洗工具）
└── references/
    ├── research/（01-writings through 06-timeline）
    └── sources/（books/transcripts/articles）
```

**关键规则**：所有调研必须存储在skill目录内部，使skill自包含可独立分发。

### Phase 1: 并行多源采集
启动6个Agent，每个负责不同维度：

| Agent | 目标 | 输出文件 |
|-------|------|---------|
| 1 著作 | 书籍、长文、论文 | 01-writings.md |
| 2 对话 | 播客、长视频、采访 | 02-conversations.md |
| 3 表达 | 社交媒体、碎片表达 | 03-expression-dna.md |
| 4 他者 | 他人评价、批评、传记 | 04-external-views.md |
| 5 决策 | 重大决策、转折点 | 05-decisions.md |
| 6 时间线 | 完整人物时间线 | 06-timeline.md |

**本地语料优先模式**：先分析用户提供素材 → 识别缺口 → 定向补充网络搜索。

**信息源黑名单**：知乎、微信公众号、百度百科。中文渠道仅接受权威媒体（36氪、晚点、财新等）和B站原始视频、小宇宙播客。

### Phase 1.5: 调研质量检查点
暂停展示调研质量摘要（来源数量、关键发现、矛盾点、不足维度），用户确认后才进入Phase 2。

### Phase 2: 框架提炼（Synthesis）

**2.1 心智模型提取**（3-7个）
- 三重验证：跨域复现（≥2领域出现）、生成力（能推断新立场）、排他性（非通用）
- 三重通过→心智模型；1-2重→决策启发式；0重→丢弃
- 每个模型记录名称、一句话描述、来源证据、应用方式、局限性

**2.2 决策启发式**（5-10条）
可表述为「如果X则Y」的快速判断规则，有具体案例支撑。

**2.3 表达DNA分析**
句式偏好、词汇特征、节奏感、幽默方式、确定性表达、引用习惯。

**2.4 价值观与反模式**
核心价值排序、明确反对的行为/思维、价值观间的内在冲突。

**2.5 智识谱系**
此人受谁影响→影响了谁→思想地图位置。

**2.6 诚实边界**
必须明确的局限：无法预测全新问题、无法替代创造力、言行可能有差距、信息截止时间。

### Phase 2.5: 提炼确认检查点
展示提炼摘要供用户确认。

### Phase 3: Skill构建
填充标准SKILL.md模板，包含：
- Frontmatter（description含来源数量、模型数量、触发词）
- 角色扮演规则
- **回答工作流（Agentic Protocol）**：问题分类 → 人物式研究 → 人物式回答
- 身份卡（50字自我介绍）
- 心智模型（含证据和局限）
- 决策启发式（含场景和案例）
- 表达DNA（转为角色扮演风格规则）
- 时间线、价值观、智识谱系、诚实边界、调研来源

**关键突破**：回答工作流让人物Skill从「鹦鹉学舌」升级为「可靠思维顾问」——遇到需要事实的问题先做功课再发言，而非凭训练数据编造。

### Phase 4: 质量验证

三项独立测试（子agent执行）：
- **已知测试**：3个公开表态问题，对比实际立场
- **边缘测试**：1个未公开讨论的相关问题，推断结果应显示不确定性
- **风格测试**：100字分析，判断辨识度

**通过标准**：
- 心智模型3-7个+每个有来源证据
- 每个模型的失效条件明确
- 表达DNA读100字能认出是谁
- 诚实边界≥3条具体局限
- 至少2对内在矛盾
- 一手来源占比>50%

### Phase 5: 双Agent精炼（后置工序）

**Agent A**（auto-skill-optimizer视角）：8维度结构评估，干跑3个测试，输出最弱2维度的改进建议。
**Agent B**（skill-creator视角）：评审触发条件完整性、角色规则可操作性、缺失关键信息。
主Agent综合两份报告，应用不冲突的改进。

## 特殊处理

### 更新已有Skill
仅启动Agent 2（对话）+ Agent 5（决策）+ Agent 6（时间线），增量更新而非重写。

### 主题Skill vs 人物Skill
主题需先定边界和流派，搜索3-5个核心人物，提取领域共识+各家分歧。

### 中文人物信息获取
B站原始视频、小宇宙播客、权威媒体采访优先。永不使用知乎、微信公众号、百度百科。

### 冷门人物（可用来源<10条）
告知用户质量受限，心智模型减至2-3个标注推测性，扩大诚实边界篇幅。

## 品味守则

| 原则 | 内涵 |
|------|------|
| 长文>金句 | 3000字essay比50条推文更揭示思维结构 |
| 争议>共识 | 最被争议观点最能揭示独特性 |
| 变化>固定 | 改变立场的地方比一直坚持更有信息量 |

**绝不做的事**：编造未说过的话、包装通用道理、忽略负面评价、信息不足时强行生成。
